# 苏州大学本科毕设论文格式转换（md → docx）

Markdown作为一种便捷的文档编写格式，不仅可以编辑论文内容，还可以接入图表，编译公式，还可以使用如Codex等工具直接对内容进行修改，这是latex（overleaf）以及传统Cord难以达到的，但我们的毕业论文的要求依旧是使用Word排版。

为了体验Markdown的便捷的同时拒绝手动排版Word的痛，本项目应运而生。

本项目可将 Markdown文档转换为带论文式排版的 Word（.docx），支持：

- 论文结构：封面（标题）、中文摘要、英文摘要、目录、正文分章分节
- 数学公式：行内 `$...$` 与块级 `$$...$$`，支持 `\tag{}` 的公式编号排版
- 表格：Markdown 表格 → Word 表格；支持表题“表x：...”；正文中的“表x”引用会随重编号同步修正
- 代码块：带行号渲染；支持清理形如 `123→` 的行号前缀；对 Python/Bash 做基础高亮
- 图片：本地图片自动缩放到页宽；配合“图x：...”作为图题时按“图 + 图题”居中排版


## 项目文件说明

- `README.md`：项目说明与排版/写法约定
- `requirements.txt`：Python 依赖列表
- `run.py`：命令行入口（convert）
- `md_to_docx.py`：核心转换实现（Markdown AST 渲染、目录/页眉页脚/页码、公式/表格/代码/图片等）
- `word_format_config.py`：排版参数集中配置（带中文注释）
- `demo_arch.png`：示例论文中的方法框架图图片素材
- `示例论文.md`：示例论文 Markdown（覆盖论文结构、公式、表格、图片、文献引用、致谢、附录代码块等）
- `示例论文.docx`：由 `示例论文.md` 导出的示例 DOCX（产物文件，用于对照检查）
- `苏州大学本科毕业设计装订及打印格式（2020版）.pdf`：格式要求参考文件

## 环境与依赖

- Python 3.10+（建议 3.11/3.12）
- 依赖见 `requirements.txt`
- 可选：Node.js（若安装，会优先使用 KaTeX 将 LaTeX 转 MathML；否则回退到纯 Python 的 `latex2mathml`）

安装依赖：

```bash
pip install -r requirements.txt
```

## 快速开始

将 Markdown 转换为 DOCX：

```bash
python run.py convert -i 示例论文.md -o 示例论文.docx
```


## 排版格式说明

完整格式说明参考`苏州大学本科毕业设计装订及打印格式（2020版）.pdf`。

本项目格式配置如下：

- 标题
  - 中文标题：小二（18pt）黑体
  - 外文标题：小二（18pt）Times New Roman
- 摘要
  - “摘要”二字：黑体小三（15pt）加粗、居中（视为章节标题）
  - 摘要内容：小四（12pt）；中文宋体、外文 Times New Roman；行距 1.5 倍；段前/段后 0 磅；首行缩进约 2 字符（0.74cm）
- 正文
  - 中文：宋体；外文：Times New Roman
  - 字号：小四（12pt）
  - 行距：1.5 倍；段前/段后：0 磅
  - 首行缩进：约等于“小四 2 字符”的常用折算（配置为 `0.74cm`）
- 目录
  - “目录”二字（目录页标题）：黑体小三（15pt）加粗、居中（视为章节标题）
  - 条目字号：四号（14pt）
  - 章题目：四号黑体；节题目：四号宋体
  - 页码连接：使用制表位 + leader dots（“……”）+ 页码域（PAGEREF）
- 页面设置
  - 页边距：上 3.3cm，下 2.7cm，左 2.5cm，右 2.5cm
  - 左侧装订线：0.5cm
- 页眉页脚
  - 页眉距顶端 2.6cm，页脚距底端 2.0cm
  - 页眉居中，小五（9pt）宋体：“苏州大学本科生毕业设计（论文）”，带横线
  - 页脚页码居中

具体字段说明直接写在配置文件每一行的中文注释中。

**注意**：有部分配置属于自作主张，原因是`苏州大学本科毕业设计装订及打印格式（2020版）.pdf`并未说明清楚，我按照个人喜好所调整，以下将详细说明：
1. “摘要”二字和“目录”二字本身在文件中未具体说明格式，这边统一按照章节标题处理。
2. ”章节题目、每节题目与正文之间空一个标准行“，这边为了美观调整为让每个标题，章节标题和正文以及页顶之间都空一个标准行（正文和页顶不空）。
3. 转换为word之后，目录中不显示页码，通过ctrl+A全选全文，再按下F9则会出现页码。
4. 参考文献格式以及文中引用请自行处理梳理好序号，按照”同一处引用多篇文献时，应将各篇文献的序号在中括号内全部列出，各序号间用英文逗号隔开，如遇连续引用序号，起讫序号间用短横线连接。多次引用同一文献时，文后参考文献列表中只需要出现一次。“在markdown的正文中使用中括号列出即可，例如：[1-3],[1,6,8]，会自动转换为上标引用。
5. 图片，表格会和相应的图注，表注绑定视为整体居中排版，由于采用流式排版，所以图片和表格会和正文内容保持一致的排版位置，若Word出现了大片留白，可以在markdown中适当调整一下位置。
6. 无论是图表还是文献引用都未设置自动编号索引（就是类似latex中的tag索引自动编号，我们并未增加该功能），只设置了纯文字展示，需要用户自行在markdown中对齐编号。
   
## 自定义排版

直接修改 `word_format_config.py` 中各字段默认值。




## Markdown 结构约定

转换器会从 Markdown 的一级标题（`#`）中拆分出各部分：

1. 第 1 个 `# ...`：中文标题（用于封面页）
2. 之后出现的 `# 摘要`：中文摘要内容（直到下一个一级标题）
3. 若存在英文标题 + `# Abstract`：英文摘要内容（直到下一个一级标题）
4. 剩余内容：正文（会生成目录，并按 `#`/`##` 构建“章/节”目录条目）

实践直接参考 `示例论文.md` 的结构。

## 图片与图注

当你想插入图片与图注时，只需要在图片后紧跟图注即可，转换后的图片和图注会视作一个整体并居中排版。

markdown写法示例：

```md
![图1](demo_arch.png)

图1：示意图图注。
```
详细参考 `示例论文.md` 中的图片图注示例。

## 表格与表题规则

- Markdown 表格会转换为 Word 表格。
- 若表格后紧跟“表x：...”表题，则会按出现顺序对表格自动编号，并同步修正正文中“表x”的引用文本。
- 详细参考 `示例论文.md` 中的表格示例。

## 公式支持

- 行内公式：`$E=mc^2$`
- 块级公式：`$$ ... $$`
- 若块级公式中包含 `\tag{}`，会按“居中公式 + 右侧编号”的方式排版。
- 详细参考 `示例论文.md` 中的公式示例。
